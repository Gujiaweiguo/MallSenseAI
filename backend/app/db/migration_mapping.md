# Legacy ŌåÆ Platform Migration Mapping

All migration jobs must support `--dry-run`, produce a report of planned inserts/updates/conflicts, and never delete or overwrite legacy source files.

## `camera_configs.json` ŌåÆ `cameras`

| Legacy field | Target field | Rule |
|---|---|---|
| `location` | `name` | Initial camera display name. If duplicated, suffix deterministic index in dry-run report only when needed for operator display. |
| `location` | `location` | Preserve original text exactly, even when duplicated. It is not a key. |
| `ip` | `ip` | Required. Combined with `port` is the natural migration identity. |
| `port` | `port` | Default to `80` if missing. |
| `username` | `username` | Preserve username. |
| `password` | `password_hash` | Never persist plaintext. Hash during real-run using platform password hashing; dry-run reports `password_present=true`. |
| n/a | `status` | Default `active`. |
| n/a | `id` | Auto-increment integer generated by DB. |

### Camera conflict resolution

1. Existing `(ip, port)` row: update mutable fields only when values differ and report `camera_update`.
2. Same `location`, different `(ip, port)`: insert distinct camera and report `duplicate_location`.
3. Missing IP: skip row, report `invalid_camera_missing_ip`.
4. Plaintext password cannot be hashed: skip real-run row, report `password_hash_failed`.

## `config.py ALARM_CONFIG` ŌåÆ `rules`

Create default rule templates per camera after camera import.

| ALARM_CONFIG keys | Rule target | Rule type |
|---|---|---|
| `threshold`, `diff_threshold`, `min_contour_area`, `area_ratio_threshold`, `min_area_ratio`, `max_area_ratio`, `max_single_object_area` | `rules.config` | `obstruction_area` |
| `interval_minutes`, `static_wait_minutes`, `min_stay_frames` | `rules.config` | `obstruction_duration` |
| `enable_yolo_detection`, `enable_image_comparison`, `detection_mode`, `yolo_weight`, `image_weight`, `human_filter_enabled` | `rules.config.detectors` | both obstruction templates |

Duplicate Python dict keys in `ALARM_CONFIG` are resolved by Python at import time; the migration records the effective value and reports `duplicate_key_effective_value` for known duplicate keys (`max_brightness`, `min_std`). Rules default to `enabled=true`, `priority=100`, and `roi_id=NULL` until ROI-specific policies are configured.

## `config.py WECHAT_CONFIG` ŌåÆ notifications

1. Insert or reuse `notification_groups(name="Legacy WeCom Default")` with empty `members` and `enabled=WECHAT_CONFIG.enabled`.
2. Insert `notification_channels(group_id, channel_type="wecom", config={"webhook_url_secret_ref": "LEGACY_WECOM_WEBHOOK_URL"}, enabled=...)`.
3. The real webhook URL must be moved to an environment/secret store. Dry-run reports whether a webhook exists but never prints it.

Conflict rules: same group/channel exists ŌåÆ update `enabled`; never overwrite an existing secret reference unless explicitly requested.

## `alarm_images/<location>/base_image.jpg` ŌåÆ `scenes.baseline_image_path`

For each camera, create a default scene named `<camera.name> default scene`. Legacy baseline files migrate to canonical path `data/assets/cameras/{camera_id}/baseline.jpg`; `scenes.baseline_image_path` stores that canonical path. If the legacy file is missing, create the scene with `baseline_image_path=NULL` and report `missing_baseline`.

Conflict rules: existing scene for camera/name ŌåÆ update baseline path only if target asset exists and checksum changed; location collision ŌåÆ mark ambiguous as described above.

## `alarm_images/<location>/safe_zones.json` ŌåÆ `rois`

Legacy format: `[{"type": "polygon", "data": [[x, y], ...]}]` in 1000x750 pixel space for most runtime paths. The importer converts to:

```json
{
  "type": "polygon",
  "points": [[0.1, 0.2], [0.3, 0.4]],
  "source": {"width": 1000, "height": 750, "path": "alarm_images/.../safe_zones.json"}
}
```

Rect zones use normalized `{ "type": "rect", "x": 0.0, "y": 0.0, "width": 1.0, "height": 1.0 }`. `normalized_coords=true` always. Degenerate polygons where all points are `[0,0]` are skipped by default and reported as `degenerate_roi`; operator can import them disabled in a later workflow if needed.

Coordinate conversion: divide x by source width and y by source height, clamp to `[0.0, 1.0]`, preserve point order, require at least three non-identical polygon points or a positive rect size.

## `alarm_images/<location>/detection.log` ŌåÆ `detection_events`

Parse each log line for timestamp, detector label, confidence/score if present, evidence filename if present, and free-form remainder. Target fields:

| Parsed data | Target |
|---|---|
| location directory | `camera_id` via location/(ip,port) mapping; ambiguous collisions reported |
| detector token | `detector_type` (`image_compare`, `yolo`, `blue_box`) |
| score/confidence | `confidence` |
| image path | `evidence_path` canonicalized under `data/assets/alerts/{alert_id}/evidence.jpg` when linked to an alert, otherwise preserved as migrated evidence asset path |
| raw line / parse warnings | `metadata` |
| timestamp | `detected_at` |

Unparseable lines are skipped in real-run unless `--include-raw-events` is set; dry-run counts them as `unparsed_detection_log_line`.

## Global conflict policy

- Idempotency key order: stable DB IDs after insert; migration matching by `(ip, port)` for cameras, `(camera_id, scene name)` for scenes, `(scene_id, source path, source index)` for ROIs, and `(camera_id, detected_at, detector_type, evidence_path)` for detection events.
- Assets are copied only when checksum differs; existing canonical files are not overwritten without a report entry.
- All destructive operations are out of scope for T01/T13 import scripts.
- Dry-run output must include counts for insert/update/skip/conflict and a machine-readable JSON section.
